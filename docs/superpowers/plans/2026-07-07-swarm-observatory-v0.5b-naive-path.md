# v0.5b — the naive path's destination

**Branch:** `dev/v0.5b` (base: main @ 3c2f60c)
**Source:** v0.5a critic-adjudicated design docket (5 items). Dossier: `.superpowers/sdd/dossier-v05b.md` (all file:line facts below are ground-truthed there against 3c2f60c).
**Thesis:** the naive path (no tour, no selection) inherits the tour's stagecraft. Every change is camera/render only.

## Pinned constraints (violating any of these fails the branch)

- **Rest-on-final-state is untouched.** The playhead rests at `tick === tickCount` (`transport.ts:21`, `transport.test.ts:50-57`). Nothing in this plan moves the playhead, the transport, or provenance semantics. The finale is stage dressing on an unchanged rest.
- **Click grammar holds:** camera moves are explicit — F focus, play, or (new, still play-scoped) the establishing/finale framings. Plain click still selects, never scrubs (`smoke.spec.ts:145-154`). Selection still gates follow.
- **Spec §8:** no React state writes in frame loops (the Timeline rAF transport batch at `Timeline.tsx:148` is the standing sanctioned writer and the natural-end edge lives inside it); zero per-frame allocation; module scratch; getState() reads.
- **Out of scope:** pulse verdict-palette semantics (`PULSE_TRUE/PULSE_FALSE`, backlog), 1x-anchor help phrase (protected — no speed-surface changes in this plan), URL schema (finale/establishing state is ephemeral display state, never serialized).

## Design rulings (adjudicated pre-plan, with dossier evidence)

1. **e0 finale is a spine rest-hold, NOT a trailHold reuse.** e0 has `trail.count === 0`; `trailHold.lit` has no visible effect and the frame-consume guard no-ops (`Scene.tsx:404`, dossier Q3). e0's journey is the ChainSpine, today selection-gated (`Scene.tsx:675`). Ruling: at natural end on a positionless run, render the FULL spine unselected in a uniform quiet rest tone (accent-family, distinct from the role-lensing triplet at `Scene.tsx:654-656`; calibrated on screenshot evidence — intent: "the complete causal record, present, not shouting"), and frame it via `frameFor(boundsFromPositions(spine positions), capped opts)` (helix radius ≈4 fits well inside the 2.5× cap). Selection during finale re-lenses the spine through the existing role machinery (selection supersedes rest tone). The DOM honesty chip (`App.tsx:253-255`) keeps carrying the presentational-layout claim.
2. **f1 finale framing is compose-around-head, NOT fit-the-corridor.** `TRAIL_FRAME_OPTS.maxDistanceFactor = Infinity` would ease to a ~340u wide shot (drone sub-pixel — dossier Q3); the capped fit falls back to a composed default around the trail center that strands the drone off-frame (dossier Q4/Q9). Ruling: new pure `finaleFraming(head, opts)` in `camera.ts` — target = head (+lift), camera = head + composed house-octant direction × FINALE_DISTANCE (≈2-2.5× the default offset length; calibrate on screenshots), so the resting drone is the subject and the hold-lit corridor recedes into the 30→400 fog for depth. `trailHold.lit = true` half of the machinery IS reused (journey stays lit at rest).
3. **Unselected play gets a true establishing shot.** On the rising edge of `playing` with no selection, no tour, and non-null `bounds`, ease to the full-fit framing of the whole trajectory (`bounds` already computed at load for free — `Scene.tsx:894-895`, dossier Q4). The run then plays out INSIDE the frame over its ~8s. This is grammar-consistent (play is an explicit camera mover, same as coast). e0/f0 (`bounds === null`) → no-op; their default framing already stages them.
4. **Frame-request channel splits framing from lighting.** `requestTrailFrame()` couples `stamp++` with `trailHold.lit = true` (`Scene.tsx:79`). The establishing shot must frame WITHOUT lighting (hold-light at play start would defeat the comet, and the rising-edge play clear at `Scene.tsx:823` would race it). Refactor: the frame-request channel carries an explicit intent (`establish | finale-trail | finale-spine`, or a lit boolean + opts selector); tour arrival keeps its exact current semantics (`TRAIL_FRAME_OPTS` + lit). All existing tour behavior must be pixel/test-identical after the refactor.
5. **Finale state lives in the zustand store, set inside the existing transport batch.** The natural-end edge is `Timeline.tsx:155` (`a.done && s.playing && !isTourActive()` — fires exactly once, dossier Q2). Finale must drive BOTH frame-loop consumers (camera, head paint, trail lit) and React consumers (ChainSpine rebuild — it has no useFrame and rebuilds via effect). A module channel can't re-render ChainSpine. Ruling: `finale: boolean` in the view store, written at the natural-end edge inside the same rAF batch that already writes `playing` (the standing transport exception; one-shot, not per-frame). Frame loops read it via `getState()` (§8-clean). **Clear grammar:** any playhead move (play, scrub, J/L step) or run-switch clears finale; selection and orbit-drag do NOT (inspecting the rest state over the lit journey is the point; drag cedes the camera ease as usual). Deep-linking directly to `?t=<tickCount>` does NOT fire the finale (natural-end is a play-path edge) — accepted asymmetry, note in code.
6. **dt-normalized easing via the exponential form.** Exactly three hand-lerp constants are FPS-dependent (dossier Q5): `0.15` focus (`motion.ts:17`, consumed `Scene.tsx:371-374` and trail-frame `428-431`) and `0.05` follow (`motion.ts:21`, via `followPan` `Scene.tsx:480`). Replace factor constants with per-second rates and `factor(dt) = 1 − exp(−k·dt)`; calibrate k for 60fps equivalence (k = −ln(1−f₆₀)·60 → ≈9.75/s for 0.15, ≈3.08/s for 0.05). Reduced-motion keeps factor = 1 (snap) — bypass, don't approximate. OrbitControls damping is three.js-internal — leave. `useFrame` delta is available at every site (2nd arg, currently unused).
7. **Follow centering: aim between head and revealed-trail midpoint.** Cause (dossier Q6): target = head with a fixed composed octant, so the one-sided trail projects off-center. Ruling: follow aim = lerp(head, revealed-midpoint, FOLLOW_TRAIL_BIAS ≈ 0.25-0.35) where revealed-midpoint is a direct read of the trail positions buffer at index ⌊tick/2⌋ (zero-alloc, no per-frame scan). Head ends ~⅓ from the leading edge; the traveled path balances the frame. Constant calibrated on screenshot evidence.
